This application helps you submit project fixes. Follow these steps carefully:

Procedure:

  1. Find the issues: Look for comments in your code marked with // DJH:
  2. Make the fixes: Address the issues indicated using the zyBooks IDE
  3. Retest carefully: Make sure your fixes work correctly
  4. Annotate your fixes: Mark each issue as fixed with // YOUR INITIALS: fixed, with an optional brief comment about the issue. Place this on the same line or the next line after the original comment. One "fixed" comment per issue comment.
  5. Download: Download the source files you changed from zyBooks
  6. Select your project: Choose the project number you're fixing from the dropdown menu on this app
  7. Upload here: Upload only your .cpp and .h files to this app
  8. Submit: Review the detected fixes and click Submit

Please Note:

  • Only fixes that are properly annotated and submitted on time will be reviewed
  • Fixes that break the code might generate negative credit
  • Late fixes may be considered for credit at the end of the quarter
